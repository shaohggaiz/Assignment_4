Question-2: What is the difference between innerHTML, innerText, and textContent ?

In JavaScript, "innerHTML" retrieves the complete HTML content within an element, including tags, while "innerText" only returns the visible text content of an element, ignoring HTML tags and hidden elements, and "textContent" retrieves all text content within an element, including hidden text, but without considering HTML tags or styling; essentially, "innerHTML" gives you the full HTML structure, "innerText" gives you the visible text on the page, and "textContent" gives you all the text in the DOM, regardless of visibility.

Question-3: What is event delegation in the DOM?

Event delegation in the DOM is a JavaScript design pattern where you attach a single event listener to a parent element, instead of adding listeners to multiple child elements, and then use the event bubbling mechanism to identify which child element triggered the event

.

Question-4: What is event bubbling in the DOM?

Bubbling in the DOM is an event propagation method. It starts with an event triggering on the innermost element and then proceeds to trigger the same event on its parent element and then its parent's parent element and so on.

.

To add, remove, and modify HTML elements using JavaScript, you can use methods like createElement , appendChild , removeChild , and properties like innerHTML and textContent .
